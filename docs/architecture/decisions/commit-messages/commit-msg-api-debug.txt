fix: resolve FastAPI validation error in /agents/state endpoint

## Summary
Fixed a critical FastAPI validation error that was preventing the `/agents/state` endpoint from returning data. The issue was caused by a type annotation mismatch between the function signature and actual return value.

## Problem
The `/agents/state` endpoint was returning empty responses due to:
```
fastapi.exceptions.ResponseValidationError: 1 validation errors:
  {'type': 'list_type', 'loc': ('response',), 'msg': 'Input should be a valid list', 'input': {'agents': [...], 'summary': {...}}}
```

## Root Cause
Function signature declared `List[Dict]` return type but implementation returned a dictionary structure:
```python
# Function signature (incorrect)
def get_agents_state() -> List[Dict]:

# Actual implementation (correct structure)
return {
    "agents": all_agents,
    "summary": {
        "total_agents": len(all_agents),
        "tier0_agents": len([...]),
        "legacy_agents": len([...]),
        "active_agents": len([...]),
        "timestamp": time.time()
    }
}
```

## Solution
Updated the function signature to match the actual return type:
```python
# Fixed signature
def get_agents_state() -> Dict:
```
